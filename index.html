Overview

What this project does

This is a continuous deployment pipeline that turns a single git push into a live website update — with no manual steps in between. When code lands on the main branch, GitHub notifies Jenkins through a webhook. Jenkins connects to a cloud server over SSH, pulls the latest code, and publishes it to an Nginx web server.

The earlier version of this project ran entirely on a local virtual machine. This version graduates it to the cloud: the deployment target is now an AWS EC2 instance with a real public IP, so the site is reachable on the open internet and webhooks reach Jenkins directly — no tunneling workarounds needed.

The goal was to learn how the core pieces of a deployment workflow fit together: source control as the trigger, an automation server as the engine, secure SSH as the bridge, and a cloud web server as the destination.
